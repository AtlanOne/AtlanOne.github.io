用Github+hexo搭建免费的个人静态博客
发表于：2021-12-17 |
字数统计: 1.8k | 阅读时长: 6分钟 | 阅读量：

前言

我一直觉得，在偌大的网络世界中有一片独属于自己的空间是一件非常酷的事情，但是自己搭建一个网站太麻烦，而且又需要花钱维护，不是一个适合大部份人的方式；作为一名资深白嫖党，这里向大家介绍一种基于github page和hexo框架的免费搭建个人静态博客的方法。

注册github账号

首先，你需要一个github账号，可以去github官网上注册，但由于是国外的网站，所以速度可能会很慢，甚至加载不出来。

可以尝试多加载几次，如果依然失败可以尝试这里面的解决办法：解决 Github 无法登录的问题,但是这个方法也不是一劳永逸的，可能隔不了多久就会失灵，可以再用ipconfig /flushdns命令多刷新一下DNS或者自行百度。

创建新的代码仓库

当你能够顺利注册并登录github后，就会看到下图这样一个页面，点击左上的Create Repository新建一个代码仓库

起名时一定要是 你的github名.github.io 的形式，也就是左边的owner下所显示的名称，不然后续可能匹配不了。Description是仓库描述，可写可不写，然后勾选public类型和增加一个README文件避免仓库为空，然后点击创建。

创建后打开，选择右边的Settings选项查看生成的gihub page，生成的这个链接就是我们博客的网页地址。

安装git

github方面处理好之后，我们就要处理电脑端的事项了。首先就是安装git，可以直接在这里选择对应的型号下载。下载安装过程很简单，这里就就不赘述了。下载安装之后可以在cmd窗口输入

git --version

查看git安装是否成功。出现版本号说明安装成功。

安装nodejs

因为hexo是一款基于nodejs的静态博客框架，所以我们还需要安装nodejs。点击这里下载。

安装过程也比较简单，这里也先带过。

安装hexo

接下来就可以安装hexo了，可以通过命令行的形式下载。这里介绍npm安装方式；首先在自己想要放置博客文件的地方新建一个文件夹，用于存放所有的博客资源，比如我这里是test文件夹，并选择该文件夹，右键选择Git bash here选项

然后在弹出的命令窗口中输入下载安装命令：

npm install hexo-cli -g

然后接着输入：

npm install hexo --save

两者的结果应该是这样的：

然后我们也可以接着输入

hexo -v

查看hexo安装是否成功。如图说明安装成功：

博客页面初访问

成功之后，我们就可以试着在本地访问hexo博客页面了。

首先在窗口中输入：

hexo init

初始化一下hexo，让它下载所需的各种配置文件，这个可能要等较久的时间，而且test文件夹（及你创建的存放博客资源的文件夹）内不能有内容，否则会失败报错。当出现下图的时候就说明初始化成功了。

此时再输入

hexo s

生成一个本地的hexo博客模板，http://localhost:4000 就个就是链接地址。

复制地址并在浏览器中打开，就会看到这样的一个hexo的博客样式模板。

tips:这里使用的本地服务器端口是4000，如果你的电脑上装了福昕阅读器该端口会被占用，可以用命令

hexo s -p 5000

将其修改为5000的端口。

这一步也可以用于以后写博客时的预览。

这步完成之后，我们两大部分就都差不多了，接下来就是将两者联系起来了。

设置ssh密钥

在命令行输入：

ssh-keygen -t rsa -C "你的Github注册邮箱"

生成密钥文件，中途可能会让你输入三次内容，依次是确认密钥文件的储存位置（一般是在C盘用户目录下新建），二是设置密码（不想设可以直接回车），三就是确认密码，一般直接都回车就行。

结果如图：

然后打开所建的.ssh文件，用记事本方式打开并复制其中id_rsa.pub的所有内容：

然后打开自己的github主页，选择右上的头像下拉菜单，选择settings选项，

然后选择里面的SSH and GPG keys选项，选择新建一个SSH key，将复制好的密钥内容粘贴到key下的框中，点击Add SSH key完成创建。

建好后就是这样的：

然后再输入：

ssh -T git@github.com

查看密钥是否和邮箱链接成功。如图，如果出现了hi 后面接的是你的用户名说明建立成功。

部署博客

创建好SSH密钥后，就可以把博客文件部署到github上发表了，打开你博客资源的根目录，打开站点配置文件_config.yml，找到deploy配置项，进行如下配置：

deploy:
  type: git
  repo: git@github.com:你的github用户名/你的github用户名.github.io
  branch: main

tips:注意，这里每一个：后面都要加一个空格，不然配置不了。另外可以直接选择用记事本打开修改

还可以对第一项#site进行相关修改：

然后在命令行内输入：

npm install hexo-deployer-git --save

获取git服务，这一步很重要，不然没办法部署。

安装了git服务之后，在命令行依次输入：

hexo clean
hexo g
hexo d

进行部署，出现INFO Deploy done: git说明部署成功。

tips:hexo clean 是清除你之前hexo生成的东西（已部署的不会）

​ hexo g 及hexo generate的简写，生成一个静态网页

​ hexo d 及hexo deploy的简写，部署文章，可以将两步写在一起 hexo d -g

部署成功之后，我们的博客就搭建成功了！https://你的github名.github.io就是你的博客网址，刚开始的博客主题是默认的hexo主题：

结语

默认的hexo主题不是特别好看，我们也可以在hexo主题官网上选择自己喜欢的主题，并按照主题发布者的配置方法去设置就好了。另外，博客文章需要用Markdown格式来写，语法很简单，几分钟就就能学个大概，写文章也需要专门的markdown文本编辑器，现在好用的非常多，软件我推荐Typora，风格简约功能齐全，可以在这里下载，在线的编辑器我推荐马克飞象，可以预览文章显示效果，并支持md格式导出下载，灰常方便。

以上就是用github+hexo搭建个人博客的全部内容，当然也可以用gitee+hexo搭建，方法大差不差，只是gitee是国内的，创建时需要实名认证，比较麻烦。还有更多有趣的玩法就靠自己去发现咯。
